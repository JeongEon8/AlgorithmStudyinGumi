# [백준 - 실버 4] 1059. 좋은 구간 

## ⏰**time**
5분

## :pushpin: **Algorithm**
정렬 + 수학(구간 개수 계산)

## ⏲️**Time Complexity**
$O(L\log L)$

## :round_pushpin: **Logic**
1. 집합 S의 원소 `L`개를 입력받아 정렬한다.
2. 어떤 수 `n`이 **집합에 포함되어 있으면** 좋은 구간을 만들 수 없으므로 정답은 `0`.
3. `n`이 포함되지 않으면, 정렬된 수열에서 `n`이 들어갈 위치를 찾아
   * `n`보다 작은 가장 큰 값 = `v[i-1]`
   * `n`보다 큰 가장 작은 값 = `v[i]`
4. 좋은 구간의 후보 범위는
   * 시작 가능 최소 `s = v[i-1] + 1`
   * 끝 가능 최대 `e = v[i] - 1`
     (이 범위 안에는 집합 원소가 없음)
5. `n`을 포함하는 좋은 구간 `[a, b]`의 개수는
   * `s <= a <= n`
   * `n <= b <= e`
   * 단, `a < b` (길이 0인 구간 제외)
6. 따라서 개수는 다음과 같이 계산된다.

[
(n-s+1)(e-n+1) - 1
]

코드에서는 동치 형태로 계산:
[
(n-s)(e-n+1) + (e-n)
]

```
정렬 후 n의 양 옆 경계 찾기:
  s = (n보다 작은 최대값) + 1
  e = (n보다 큰 최소값) - 1

if n이 집합에 있으면 0
else 정답 = (n-s+1)*(e-n+1) - 1
```
