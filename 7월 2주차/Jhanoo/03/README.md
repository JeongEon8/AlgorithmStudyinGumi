# [PGS - Lv2] 03_마법의 엘리베이터

## ⏰**time**

14분

## :pushpin: **Algorithm**

* Greedy
* 수학
* 자릿수 처리

## ⏲️**Time Complexity**

$O(D)$
`D`는 `storey`의 자릿수이다.
각 반복에서 일의 자리를 처리한 뒤 `storey /= 10`으로 다음 자리로 이동하므로, 모든 자릿수를 한 번씩 확인한다.

따라서 시간 복잡도는 $O(D)$이다.

## :round_pushpin: **Logic**

1. 엘리베이터는 `10^n` 단위만큼 위나 아래로 이동할 수 있다.

   따라서 현재 층의 일의 자리부터 차례대로 처리하면 된다.

2. 현재 일의 자리 숫자와 그다음 자리 숫자를 구한다.

   ```java
   int digit = storey % 10;
   int nextDigit = (storey / 10) % 10;
   ```

3. 현재 자리 숫자가 `5`보다 작으면 아래로 내리는 것이 더 적은 버튼을 사용한다.

   예를 들어 일의 자리가 `3`이면 `-1` 버튼을 3번 눌러 해당 자리를 `0`으로 만든다.

   ```java
   answer += digit;
   ```

4. 현재 자리 숫자가 `5`보다 크면 위로 올리는 것이 더 적은 버튼을 사용한다.

   예를 들어 일의 자리가 `8`이면 `+1` 버튼을 2번 눌러 다음 10의 배수로 만든다.

   ```java
   answer += 10 - digit;
   storey += 10 - digit;
   ```

5. 현재 자리 숫자가 `5`라면 위로 가는 경우와 아래로 가는 경우의 현재 비용은 모두 5로 같다.

   따라서 다음 자리 숫자를 확인한다.

   * 다음 자리 숫자가 `5` 이상이면 올림
   * 다음 자리 숫자가 `5` 미만이면 내림

   ```java
   if (digit > 5 || (digit == 5 && nextDigit >= 5)) {
       answer += 10 - digit;
       storey += 10 - digit;
   } else {
       answer += digit;
   }
   ```

6. 올림을 선택한 경우 `storey`에 `10 - digit`을 더해 다음 자리로 올림을 반영한다.

   이후 현재 자리를 제거하기 위해 10으로 나눈다.

   ```java
   storey /= 10;
   ```

7. 모든 자릿수를 처리할 때까지 반복하고, 누적한 버튼 입력 횟수를 반환한다.

   ```java
   return answer;
   ```

## :black_nib: **Review**

* 처음에는 각 자리에서 위로 이동하거나 아래로 이동하는 두 가지 선택을 DFS로 탐색할 수 있다고 생각했다.
* 하지만 현재 자리 숫자가 `5`보다 작은지 큰지에 따라 더 유리한 방향이 정해지고, `5`인 경우에만 다음 자리를 확인하면 되므로 그리디로 해결할 수 있었다.
* `5`에서는 현재 자리 비용이 같기 때문에 다음 자리 숫자가 큰지 확인해 올림 여부를 결정하는 것이 핵심이었다.
* 올림을 선택했을 때 `storey += 10 - digit`으로 실제 다음 자리까지 반영한 뒤 10으로 나누는 방식이 직관적이었다.
* 모든 자릿수를 한 번씩만 확인하므로 자릿수에 비례하는 시간에 해결할 수 있었다.
