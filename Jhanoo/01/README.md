# [백준 - 골드 3] 2533. 사회망 서비스(SNS)

## ⏰**time**

25분

## :pushpin: **Algorithm**

- 트리 DP
- DFS

## ⏲️**Time Complexity**

$O(N)$  
(\(N\): 노드 수, 각 간선은 DFS에서 한 번씩만 확인)

## :round_pushpin: **Logic**

1. 트리를 1번 노드를 루트로 잡고 DFS를 돌리면서, 각 노드가 얼리어답터인지 여부에 따라 DP 값을 갱신합니다.

2. `dp[u][0]`: 정점 `u`가 얼리어답터가 아닐 때(0), `u`의 서브트리에서 필요한 최소 얼리어답터 수  
   `dp[u][1]`: 정점 `u`가 얼리어답터일 때(1), `u`의 서브트리에서 필요한 최소 얼리어답터 수

3. 점화식은 다음과 같습니다.  
   `u`가 얼리어답터가 아니면, 모든 인접 노드(자식)는 반드시 얼리어답터여야 하므로 `dp[u][0] += dp[v][1]`  
   `u`가 얼리어답터면, 자식 `v`는 얼리어답터/비얼리어답터 중 최소를 선택 `dp[u][1] += min(dp[v][0], dp[v][1])`

```java
dp[u][0] = 0; // u가 얼리어답터가 아님
dp[u][1] = 1; // u가 얼리어답터임 (자기 자신 포함)

for (int v : graph[u]) {
    if (visited[v]) continue;
    dfs(v);

    dp[u][0] += dp[v][1];
    dp[u][1] += Math.min(dp[v][0], dp[v][1]);
}
```

4. 루트(1번)의 두 상태 중 더 작은 값이 정답입니다.

## :black_nib: **Review**

- 처음엔 그리디로 리프노드의 부모를 얼리어답터로 만들고 그 부모의 부모를 얼리어답터로 만드는 방향으로 생각했다가, 트리의 구조가 어떻게 형성되는지에 따라 답이 바뀔 것이라 생각하고 dp로 풀었다.
