# [PGS - lv2] 02. 줄 서는 방법

## ⏰**time**

## :pushpin: \*\*Algorithm

순열

## ⏲️**Time Complexity**

$O(n^2)$

## :round_pushpin: **Logic**

순열을 직접 생성하면 최대 20!개의 경우를 탐색해야 하므로 시간 내에 해결할 수 없다.

1. 순열을 그룹으로 나누기

   예를 들어 `n = 4`인 경우 전체 순열의 개수는 `4! = 24`개이다.

   사전순으로 정렬하면 다음과 같이 첫 번째 숫자에 따라 그룹이 나뉜다.

   `1 _ _ _` → 3! = 6개
   `2 _ _ _` → 3! = 6개
   `3 _ _ _` → 3! = 6개
   `4 _ _ _` → 3! = 6개

   즉, 첫 번째 숫자가 하나 고정되면 나머지 숫자를 배치하는 경우의 수는 `(n - 1)!`개가 된다.

2. 현재 k가 속한 그룹 찾기

n = 4, k = 14라고 가정하자.

인덱스 계산을 위해 `k--`를 수행하여 0-based index로 변환한다.

첫 번째 자리의 그룹 크기는 `3! = 6`

따라서 `idx = k / 6`을 계산하면 `13 / 6 = 2`가 된다.

현재 사용 가능한 숫자가 `[1, 2, 3, 4]` 이므로 인덱스 2에 해당하는 숫자 3이 첫 번째 자리에 들어간다.

```java
answer = [3]
nums = [1, 2, 4]
```

3. 다음 자리 계산

첫 번째 숫자를 선택한 이후에는 해당 그룹 내부에서 몇 번째 순열인지 다시 계산해야 한다.

`k %= 6` -> `13 % 6 = 1`

이제는 3으로 시작하는 순열들 중 1번째(0-index)를 찾는 문제가 된다.

남은 숫자는 `[1, 2, 4]`이며, 각 그룹의 크기는 `2! = 2`이다.

`idx = 1 / 2 = 0` 따라서 두 번째 자리는 1이 된다.

```java
answer = [3, 1]
nums = [2, 4]
```

```java
for (int i = 1; i <= n; i++) {
    nums.add(i);
    fact *= i;
}
k--;

for (int i = 0; i < n; i++) {
    fact /= (n - i);

    int idx = (int) (k / fact);

    answer[i] = nums.remove(idx);
    k %= fact;
}
```
