# [PGS - Lv2] 01_시소 짝꿍

## ⏰**time**

17분

## :pushpin: **Algorithm**

* 정렬
* HashMap
* 누적 카운팅
* 수학

## ⏲️**Time Complexity**

$O(N \log N)$
`N`은 `weights` 배열의 길이이다.

먼저 몸무게 배열을 정렬하는 데 $O(N \log N)$이 걸린다.
이후 각 몸무게를 한 번씩 순회하면서 `HashMap`에서 짝이 될 수 있는 이전 몸무게의 개수를 조회하므로 평균 $O(N)$이 걸린다.

따라서 전체 시간 복잡도는 $O(N \log N)$이다.

## :round_pushpin: **Logic**

1. 두 사람이 시소에서 균형을 이루려면 다음 조건을 만족해야 한다.

   ```text
   몸무게1 × 거리1 = 몸무게2 × 거리2
   ```

   사용할 수 있는 거리의 비율을 정리하면, 시소 짝꿍이 될 수 있는 몸무게 비율은 다음 네 가지이다.

   ```text
   1 : 1
   1 : 2
   2 : 3
   3 : 4
   ```

2. 몸무게 배열을 오름차순으로 정렬한다.

   현재 몸무게 `w`를 둘 중 큰 몸무게로 보고, 이전에 등장한 더 작거나 같은 몸무게만 확인하기 위함이다.

   ```java
   Arrays.sort(weights);
   ```

3. 이전에 등장한 몸무게별 인원수를 저장하기 위해 `HashMap`을 사용한다.

   ```java
   Map<Integer, Integer> count = new HashMap<>();
   ```

4. 현재 몸무게가 `w`라면, 이전 사람 중 다음 몸무게를 가진 사람과 시소 짝꿍이 될 수 있다.

   ```text
   1 : 1 → w
   1 : 2 → w / 2
   2 : 3 → w × 2 / 3
   3 : 4 → w × 3 / 4
   ```

5. 먼저 같은 몸무게를 가진 이전 사람의 수를 정답에 더한다.

   ```java
   answer += count.getOrDefault(w, 0);
   ```

6. `1 : 2` 비율은 현재 몸무게 `w`가 2로 나누어떨어질 때만 가능하다.

   ```java
   if (w % 2 == 0) {
       answer += count.getOrDefault(w / 2, 0);
   }
   ```

7. `2 : 3` 비율은 현재 몸무게 `w`가 3으로 나누어떨어질 때만 가능하다.

   ```java
   if (w % 3 == 0) {
       answer += count.getOrDefault(w * 2 / 3, 0);
   }
   ```

8. `3 : 4` 비율은 현재 몸무게 `w`가 4로 나누어떨어질 때만 가능하다.

   ```java
   if (w % 4 == 0) {
       answer += count.getOrDefault(w * 3 / 4, 0);
   }
   ```

9. 현재 사람과 이전 사람들의 짝 계산이 끝난 뒤, 현재 몸무게 `w`의 등장 횟수를 증가시킨다.

   ```java
   count.put(w, count.getOrDefault(w, 0) + 1);
   ```

   현재 사람을 먼저 Map에 넣으면 자기 자신이 `1 : 1` 짝으로 계산될 수 있으므로, 반드시 계산이 끝난 뒤 등록해야 한다.

10. 모든 사람을 확인한 뒤 누적된 짝의 수를 반환한다.

```java
return answer;
```

## :black_nib: **Review**

* 처음에는 모든 사람 쌍을 직접 비교하는 $O(N^2)$ 풀이를 생각할 수 있지만, 몸무게 비율이 정해져 있다는 점을 이용하면 더 효율적으로 해결할 수 있었다.
* 배열을 정렬한 뒤 현재 사람보다 앞에서 처리한 사람만 확인해 같은 쌍을 중복해서 세지 않도록 했다.
* 이전 몸무게별 인원수를 `HashMap`에 저장하면, 현재 사람과 짝이 될 수 있는 몸무게의 등장 횟수를 바로 조회할 수 있었다.
* 같은 몸무게의 사람도 누적 개수를 더하는 방식으로 별도의 조합 계산 없이 자동으로 쌍의 수를 셀 수 있었다.
* `2 : 3`, `3 : 4` 비율을 계산할 때 정수 나눗셈으로 잘못된 값이 만들어지지 않도록 나누어떨어지는지 먼저 확인하는 것이 중요했다.
* 가능한 짝의 수가 `int` 범위를 넘을 수 있으므로 정답은 `long`으로 관리해야 했다.
